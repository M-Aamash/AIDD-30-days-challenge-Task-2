## 1. Nine Pillars Understanding

### - Why is using AI Development Agents (like Gemini CLI) for repetitive setup tasks better for your growth as a system architect?

AI Development Agents jaise Gemini CLI repetitive aur boring setup tasks ko bohot tezi se kar dete hain.
Is se humara waqt bach jata hai, aur hum sirf code likhne ke bajaye poore system ko samajhne par focus kar sakte hain.
Yani developer zyada creative aur architect-level thinking develop karta hai, jo growth ke liye bohot zaroori hai.

### - Explain how the Nine Pillars of AIDD help a developer grow into an M-Shaped Developer.

AIDD ke Nine Pillars woh tamam zaroori skills cover karte hain jo kisi developer ko M-Shaped Developer banati hain.
Isme coding, architecture thinking, AI integration, testing, deployment, data management, security, collaboration, aur problem-solving sab shamil hain.
Jab developer yeh 9 pillars seekh leta hai, to woh complex systems ko design, manage aur scale kar sakta hai.

## 3. Architecture Thinking

### How does architecture-first thinking change the role of a developer in AIDD?

Architecture-first thinking me developer sirf coder nahi rehta woh system ka blueprint designer ban jata hai.
Woh pehle sochta hai ke system ka structure kya hoga, layers kaise connect hongi, konse tools lagengay, aur AI ka integration kaise hoga.
Yani developer ka role coder architect ban jata hai.

### Explain why developers must think in layers and systems instead of raw code.

Layers aur systems me sochne se code clean, maintainable aur scalable banta hai.
Is se future me changes, updates aur bug fixes bohot aasaan ho jate hain.
Agar sab kuch raw code me ho, to project jaldi messy ho jata hai aur maintain karna mushkil ho jata hai.
